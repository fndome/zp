# game-update

游戏服务器玩家状态批量更新示例。

## 场景

200ms 一批战斗事件 → 3~8 个玩家获得奖励。不是每次直接写 DB，而是：

```
Submit(userId) → batcher 攒到 100 人或 3 秒
  → processor: 查最新 DB 状态 → 合并增量 → 一批 UPDATE
```

## 关键点

| 步骤 | 说明 |
|------|------|
| `Submit` | 只传 `userId`，极轻量，不阻塞 |
| 攒批 | `channel` 天然线程安全，无需加锁 |
| `processor` 执行时 | 查此刻最新状态（不是 3 秒前提交时的值） |
| 去重 | 同一玩家 3 秒内多次投递 → `map[userId]` 去重 |
| 写回 | 一批 `UPDATE`，IO 降 99% |

## 运行

```bash
cd example/game-update
go run main.go
```

输出示例：

```
=== 游戏服务器: 玩家状态批量更新 (xg.Batcher) ===

总提交: 1280 次 | 实际DB写入: 15 次

  玩家 1  金币:12850  HP:22  经验:4200  最后更新:20:26:11
  玩家 2  金币:11900  HP:18  经验:4320  最后更新:20:26:11
  玩家 3  金币:13400  HP:16  经验:4260  最后更新:20:26:11
```

1280 次提交 → 15 次 DB 写入 = 98.8% 的 IO 被 batcher 消灭。
